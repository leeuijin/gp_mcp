Q2. 데이터베이스의 사용자 테이블 전체를 리스트업하고  각각의 테이블과 각각의 컬럼마다  어떤 데이터를 관리하고 있는지 분석해줘
==============================================================================================================================
1. 데이터베이스 개요
데이터베이스 개요
데이터베이스명: kbc
총 테이블 수: 9개
데이터베이스 크기: 116.21 MB
분산 정책: HASH DISTRIBUTED (모든 테이블)
2. 전체 테이블 리스트 및 상세 분석
생략
3. 데이터 관리 흐름도
원자재 조달 → 생산 실행 → 품질 관리 → 출하 관리
    ↓            ↓            ↓          ↓
raw_materials  production_lot  quality_    shipments
               ↓ 설비운영      inspection  ↓
            equipment_log                customers
            lot_material_usage
               ↓
            (추적 가능성)
4. 주요 특징
| 특징 | 설명 |
|------|------|
| **분산 정책** | 모든 테이블이 HASH DISTRIBUTED로 설정되어 있음 (병렬 처리 최적화) |
| **파티셔닝** | production_lot 테이블만 production_date 기준으로 월별(12개월) 범위 파티션 |
| **시계열 데이터** | equipment_log (100만건)이 가장 큼 - 센서 데이터 관리 |
| **추적성(Traceability)** | lot_material_usage로 원자재부터 최종 제품까지 추적 가능 |
| **배터리 산업 특화** | CELL/MODULE/PACK 타입, 용량/전압 사양, 불량률 관리 등 배터리 산업 최적화 |
